# Beat the House 0.3.0 Release Checklist

This document is the source-controlled readiness ledger for the 0.3.0 source
release candidate. R11 partitioned the release-prep work into auditable commits;
the remaining release actions are manual itch.io publishing and real mobile
signing credentials. Generated reports stay under `.tmp/` or Godot `user://`
paths and are not canonical release docs.

- Target version: `0.3.0`

- Current stamped export preset versions: `0.3.0` for Windows, Android, and iOS
- Version bump owner: T7.4 Itch Publish Pipeline, committed in `c3b89ed`
- Validation runs captured: 2026-07-03
- Main scene: `res://scenes/main.tscn`
- Release framing: start screen and README state simulated gambling only, with
  no real-money wagering or cash prizes

## Included 0.3 Scope

- Pinball slot feature rework with shipped acceptance probes and post-rework
  slot acceptance evidence.
- Slot release fixes: real reel-shift nudge payout path, bonus-stuck
  completion/watchdog coverage, cabinet/acceptance gates, and deep-audit RTP
  checks.
- Skill-cheat pillar across all seven games, with 11 registered cheat actions:
  pull-tab detector scan, slot nudge, loaded toss, palmed swap, hole-card peek,
  count cards, baccarat shoe read, edge sort, wheel read, past-post, and video
  poker mark-holds.
- Tier-2 venue progression with `kitty_cat_lounge` and `delta_queen`, plus
  route costs, travel locks, and scouting.
- Seeded world map travel graph with stored node environments and modal map UI.
- Expanded Act 1 content: lenders, services, events, challenge runs, item
  synergy, and triggered event flow.
- Performance work: table resolve regression fixed, per-frame deep-copy paths
  removed, game-surface probe coverage added, and full suite green on the
  current tree.
- R9 cleanup committed in `e01aef7`: dead script/tool/doc/art cleanup,
  generated metadata untracked, icon generator compliance, and tracked
  `.import`/`.uid` validator hardening.
- Documentation truth pass committed in `f9e9b9e`: README current-state update
  and this checklist.

## Itch Page Checklist

- Title: `Beat the House`
- Short copy: `A run-based casino crime spiral about debt, heat, crooked tables, and one last climb to the Grand Casino.`
- Long copy should frame it as a single-player Godot casino roguelike with seeded runs, full table simulations, risky cheats, debt, heat, travel, and terminal Act 1 victory/failure states.
- Simulated-gambling framing: state clearly that the game has simulated gambling only, no real-money wagering, no cash prizes, no gambling monetization, and no store credentials in the repo.
- Uploads: Web zip on butler channel `html`, Windows zip on channel `windows`, both with user version `0.3.0`.
- One-time user auth step: install butler locally and run `butler login` once; credentials stay out of source control and are never handled by automation.
- Web embed settings: browser-playable file, 1280 x 720 embed, fullscreen button enabled, SharedArrayBuffer support enabled.
- Screenshots: use `branding/beat_the_house_logo.png` plus the curated set under `branding/screenshots/` (`01_start_screen.png`, environment rooms, Slot, Blackjack, Roulette, Video Poker, Baccarat, Bar Dice, and Pull Tabs).
- GIF capture list: start screen to seeded new run, world-map travel choice, slot cabinet spin/feature tease, blackjack clean hand plus cheat-risk moment, roulette spin/settle, and Grand Casino victory/showdown route.

## Known Release Limitations

- T7.3 balance is accepted for this fast release cut: the deterministic metrics
  pass and the 3-run mouse release-smoke batch replaces the original 60-run
  statistical batch. This is an explicit reduced-confidence release decision,
  not a 60-run statistical signoff.
- Publish itself remains a user action through butler or the itch.io dashboard.
- Android and iOS store submission require real signing/team credentials.

## Final Decision

Status: `0.3.0` is source-release ready after the R11 final gate. The current
tree has committed release-prep work, green visual QA evidence, deterministic
balance metrics, and a passing T7.3-FAST 3-run mouse balance smoke. For this
cut, 3 runs supersede the original 60-run balance batch. The remaining release
actions are user-side publish execution and mobile signing credentials.
