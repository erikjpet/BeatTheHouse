# 0.5 UI Overhaul — Design Brief

Status: PLANNED, runs LAST in 0.5 (owner-locked). This brief holds the
design philosophy and the open questions; phase 0 (audit + design
system) is executable now via `docs/todo/ui05_0_audit_design_system_prompt.md`.
Later phase prompts are authored AFTER phase 0's inventory report and
the owner's answers to the open questions below.

## Why last

Every other 0.5 system (casino rooms and Cage fixtures, coach bubbles,
tutorial, run report, cheat interactions) creates or reshapes UI
surfaces. The overhaul skins a finished feature set once, instead of
chasing a moving one.

## Design philosophy (proposed — owner may amend)

1. **Keep the identity, raise the craft.** The neon-pixel, drawn-not-
   asset look IS the game's brand (store page, promos, icon art all
   trade on it). The overhaul is not a restyle to a different genre of
   UI — it is consistency, hierarchy, and polish applied to the
   existing identity.
2. **One design system, everywhere.** Today's UI grew organically:
   VisualStyle palette + FoundationWidgets exist, but spacing, font
   scale steps, panel radii, focus/hover states, and button variants
   drift between screens. The overhaul's core deliverable is a single
   token + widget system (spacing scale, type scale, color roles,
   states) that every screen consumes — so future screens are
   assembled, not invented.
3. **Hierarchy over chrome.** Screens should answer "what matters right
   now" in one glance: primary action prominent, danger states loud
   (heat, deadlines, all-ins), reference info quiet. No decoration that
   costs legibility.
4. **Readable at both sizes.** Every surface must hold at 1280×720 and
   in small-screen play mode without scrolling (tabs/compaction where
   needed — the run report set the precedent).
5. **The dealer's-table feel.** Interactions should feel physical and
   immediate: press states, chip/card motion cues, consistent
   confirm-on-risk patterns. Reduce-motion strips motion, never
   information.
6. **Zero performance tax.** The overhaul obeys the zero-copy per-frame
   and idle-liveness rules; polish must not spend the web frame budget
   (perf probe surfaces stay within budgets).

## Scope (inventory to be confirmed by phase 0)

Start screen, settings, run HUD + action panels, environment/room
interaction chrome, game surface framing (per-game surfaces keep their
table identities; framing/controls unify), popups (events, wagers,
inventory, journal, map overlay, Cage room fixtures), talk dock, coach
bubbles, run report, meta home screens, collection browser, and the
career/stats view (NEW in this overhaul — profile run history +
lifetime stats finally get a surface; deferred here by owner decision).

## Open questions for the owner (answer before phase 1 prompts)

1. **Career/stats view placement**: start screen tab, meta home object
   (e.g. a scrapbook in the house), or both?
2. **Font direction**: keep the current pixel/system font mix, or adopt
   a single licensed pixel font family for everything?
3. **Controller/keyboard support**: is gamepad/keyboard-first
   navigation in scope for this overhaul or explicitly out (mouse/touch
   only)? This decision shapes every focus-state token.
4. **How bold may phase 2 get?** Per-screen layout REDESIGNS allowed
   where hierarchy demands it, or strictly reskin-in-place?

## Phases

- **Phase 0 (executable now, prompt exists)**: full UI surface
  inventory + heuristics audit; build the design-token + widget-kit
  foundation and prove it on ONE screen (settings) without touching the
  rest. Produces the surface-by-surface findings report that phase
  prompts are authored from.
- **Phase 1**: systemic sweep — migrate all shared chrome (panels,
  buttons, labels, popups) to the token system.
- **Phase 2**: per-surface passes in priority order from the phase-0
  report (run HUD and game framing first; meta screens; career/stats
  view built here).
- **Phase 3**: motion/feel pass + full gate battery + before/after
  captures for the 0.5 promo.
