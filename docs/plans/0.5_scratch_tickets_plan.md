# 0.5 Scratch-Off Tickets — Design Plan (Gas Station Casinos)

Status: PLANNED, owner-directed (2026-07-14). A NEW game module — the
eighth game — modeled on the methodology of the Steam game "Scratchy
Scratchy": a growing collection of scratcher types, each with its own
quirks, surprises, and downsides — monstrous payouts with nasty
penalties, steady earners, and duds that pay nothing. Prompts:
`docs/todo/scratch05_1_module_prompt.md` and
`docs/todo/scratch05_2_roster_items_prompt.md`.

## Owner direction (locked)

- Lives at GAS STATION CASINOS. Layout: a typical scratch-ticket
  VENDING MACHINE on the LEFT, the play surface on the RIGHT.
- Unlike pull tabs, every ticket TYPE is unique — distinct design,
  prize structure, and gimmick, inspired by real scratch tickets.
- The player ACTUALLY SCRATCHES the tickets (pointer scrubbing reveals
  cells).
- Payouts work like pull tabs: a clerk present cashes winners.
- Items interact: xray_glasses MUST have an effect here; add a few more
  item effects (new items allowed, or new effects on existing items).
- Ticket outcomes respect relevant game state (luck, drink, heat).

## Core loop

Buy a ticket from the machine's seeded stock → it lands on the play
surface with its unique face → scratch the latex (drag to scrub; per-
cell coverage threshold reveals symbols; an accessibility "scratch all"
control exists and reduce-motion implies it) → resolve per the ticket's
prize rules → winners cash at the clerk (pull-tabs redeemer pattern),
big wins draw clerk attention/heat like other games.

**Determinism contract**: a ticket's full outcome is fixed by seeded
roll AT PURCHASE. Scratching is presentation and pacing — reveal order
never changes the result. Drunk level degrades the presentation (smudged
reveals, shaky scrub), never the outcome; effective_luck shifts the
purchase-time prize roll exactly like other games' luck hooks.

## Ticket roster (launch set — all data-driven, real-ticket-inspired)

| Ticket | Price band | Gimmick |
| --- | --- | --- |
| Lucky 7s | cheap | classic match-3 sevens; gentle intro |
| Cash Cow | cheap | steady earner: high hit rate, small prizes |
| Gold Rush Doubler | mid | multiplier symbols (2x/5x/10x) on wins |
| Bonus Box | mid | key symbol unlocks a separate bonus area |
| Word Hunt | mid | crossword-style grid; lots of scratching, mid volatility |
| Second Chance | mid | some losers convert to a free cheaper ticket |
| High Voltage | expensive | monstrous top prize; SHOCK symbols each zap cash on reveal |
| Devil's Cut | expensive | big prizes, but the devil symbol takes a cut and adds heat |
| Fool's Gold | expensive | looks premium, pays almost nothing — the trap that teaches reading prize tables |
| Midnight Rare | rare stock | seldom in the machine; rare symbols grant a temporary luck buff on top of cash |

Machine stock is a seeded subset per visit/day (ties into day-keyed
rotation streams), so the collection feels discovered over time; rare
tickets appear occasionally. Per-profile "ticket types discovered"
tracks the collection (simple stat now; meta hooks later).

## Item interactions

- `xray_glasses` (mandatory): peek 2-3 unscratched cells before
  committing to scratch — surveillance-class, with the usual
  item-cheat heat implications. This is the game's cheat surface
  (like pull tabs, no distinct skill cheat — item-driven by design,
  consistent with the skill-cheat pass exemption rules).
- `tarot_card` (new effect): sense a ticket's "fortune" — a vague
  tier read (cold / warm / hot) before scratching.
- 1-2 NEW items allowed where existing ones don't fit, e.g. a "lucky
  penny" scratch tool that shields against one penalty symbol per
  ticket. All effects data-driven through the existing ItemEffect keys.
- Global luck items apply automatically via effective_luck.

## Presentation

Each ticket type gets a UNIQUE face drawn in the game's immediate-mode
pixel style (distinct palette, layout, symbol set per type — the
generated-icon pipeline's aesthetic). Machine left / surface right on
the 900×430 board; the vending machine shows current stock; the clerk
stands at the counter (redeemer pattern).
