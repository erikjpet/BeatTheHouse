# Beat the House 0.3.1 Release Checklist

This document is the source-controlled readiness ledger for the 0.3.1
performance and stability hardening release. Generated reports stay under
`.tmp/` or Godot `user://` paths and are not canonical release docs.

## Release Identity

- Target version: `0.3.1`
- Current stamped project version: `0.3.1` in `project.godot`
- Current stamped export preset versions: `0.3.1` for Windows, Android, and iOS
- Android version code: `4`
- Validation runs captured: 2026-07-04
- Main scene: `res://scenes/main.tscn`
- Release framing: simulated gambling only; no real-money wagering or cash
  prizes
- Release dependency: web-export audio playback fix committed in `6ae682b`
- SC.3 ledger/version/checklist commit: `7deb874`

## Included 0.3.1 Scope

- No new content or feature scope. 0.3.1 hardens the shipped 0.3.0 content.
- Performance hardening: idle game-surface redraws removed, per-frame
  allocation tripwires added, resolve budgets enforced, suite wall-time budgets
  enforced, and the default Godot gate now includes a fast perf smoke.
- Stability hardening: 30-run and 60-run mouse batches are clean, hostile UI
  input fuzz is covered, stuck-state sweep coverage is generalized, and long
  soak coverage asserts bounded growth.
- State integrity hardening: mutation firewall coverage, save/load interruption
  fuzz, 0.3.0 save compatibility, deterministic two-process probe, and
  read-only/zero-copy guardrails.
- Export hardening: Web and Windows itch packages were rebuilt from the 0.3.1
  version-stamped tree and hashed.

runtime gate listed above passed on the version-stamped source tree, the release
ledger is committed, and Web/Windows itch packages are built with recorded
hashes. Remaining actions are manual publishing and credentialed mobile store
